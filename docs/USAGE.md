# 사용 설명서 (Usage Guide)

e-con AR0234 4-카메라 모듈용 웹 기반 카메라 테스트 도구 사용법. 프로젝트 개요/설치는
[`README.md`](../README.md), 설계 배경은 [`docs/superpowers/specs/2026-07-06-econ-multicam-test-design.md`](superpowers/specs/2026-07-06-econ-multicam-test-design.md) 참조.

## 1. 실행

```bash
source .venv/bin/activate
python3 run.py                              # host 0.0.0.0, port 8888, 1920x1080
python3 run.py --port 8888 --width 1920 --height 1080
```

브라우저에서 `http://<jetson-ip>:8888/` 접속. 페이지를 열면 자동으로 카메라를 감지하고
"단일 촬영" 탭이 활성화된다.

## 2. 화면 구성

- **상단 헤더**
  - **해상도** 드롭다운: 모든 카메라 공통 해상도를 변경(카메라가 지원하는 해상도 목록에서 선택).
    변경 시 현재 실행 중인 스트림/동기 세션을 재시작한다.
  - **카메라 재감지**: `/dev/video0`~`15` 중 AR0234 카메라를 다시 스캔. 카메라를 새로 연결/분리했을 때 사용.
  - **프로그램 종료**: 모든 카메라 파이프라인을 정리하고 서버 프로세스를 종료한다. 이후 다시 쓰려면
    터미널에서 `python3 run.py`를 재실행해야 한다.
- **탭**: 단일 촬영 / 다중 동기 촬영 / 캘리브레이션

## 3. Mode 1 — 단일 촬영

1. 상단 드롭다운에서 카메라 1대를 선택하면 즉시 **라이브 프리뷰**(저해상도, 640×360)가 표시된다.
   조준/초점 확인용이며 실제 저장 해상도와는 무관하다.
2. **Capture** 클릭 → 그 순간 원본 해상도(헤더에서 설정한 해상도) 정지 프레임을 캡처해 크게 표시한다.
3. **Save** 클릭 → `captures/<YYYYMMDD_HHMMSS>/video<N>.jpg` 로 저장.

## 4. Mode 2 — 다중 동기 촬영

1. 탭 진입 시 체크박스로 표시된 카메라(기본 전체 선택) 기준으로 동기 세션이 시작된다.
   - 선택된 카메라 전체에 V4L2 `frame_sync=1`(30Hz)을 설정한 뒤, **하나의 GStreamer 파이프라인**
     (공유 클럭)으로 각 카메라의 라이브 프리뷰를 그리드에 표시한다.
   - 체크박스를 바꾸면 세션이 재시작된다.
2. 그리드 아래 **상태 표시줄**이 0.7초 간격으로 갱신되며 현재 동기화 상태를 보여준다.
   - 🟢 동기 양호: 최대편차 2ms 미만
   - 🟡 동기 주의: 2ms 이상 10ms 미만
   - 🔴 동기 불량: 10ms 이상
   - 표시 값: **최대편차(spread, ms)**, **표준편차(σ, ms)**, 기준 카메라(가장 이른 타임스탬프)
3. **Sync Capture** 클릭 → 선택된 카메라 전체의 원본 프레임을 동시에 여러 장씩 모은 뒤, PTS(타임스탬프)가
   서로 가장 가깝게 맞는 한 세트를 골라 캡처로 확정한다. 그리드가 캡처 결과로 고정(freeze)되고
   각 이미지 아래에 기준 카메라 대비 상대 시간차(ms)가 표시된다.
4. **라이브 재개** 클릭 → 캡처로 멈춘 그리드를 다시 라이브 프리뷰로 되돌린다.
5. **Save All** 클릭 → `captures/<timestamp>/video<N>.jpg` 전체 저장 + 동기 통계를 담은
   `captures/<timestamp>/sync_report.json` 저장.

> 육안으로 동기화를 검증하려면 움직이는 피사체(예: 회전하는 물체, 스톱워치)를 촬영해
> 각 카메라 이미지를 나란히 비교하면 된다.

## 5. Mode 3 — 캘리브레이션

체커보드/ChArUco 기반 **캘리브레이션 이미지 수집 + 검증** 도구. 실제 K·distortion·extrinsic
행렬 계산은 하지 않고, "캘리브레이션에 쓸 만한" 이미지만 걸러 저장한다(계산은 오프라인 후속 단계).

**공통 — 보드 설정** (상단 폼): 보드 종류(체커보드 기본 / ChArUco), 교차점 가로·세로 수,
칸 크기(mm)를 입력한다. ChArUco 선택 시 마커 크기(mm)와 ArUco 사전이 추가로 나타난다.
"예상 교차점 N개" 힌트로 설정을 확인할 수 있다.

- 체커보드: 가로·세로는 **내부 코너 수**(예: 10×7 칸 보드 → 9×6). 보드가 화면에 **완전히** 보여야 검출된다.
- ChArUco: 가로·세로는 **칸 수**. 보드가 부분적으로 가려져도 검출되어 다중 카메라 겹침 확보에 유리하다.

### 5.1 Intrinsic (단일 카메라)

1. 카메라 1대 선택 → 라이브 프리뷰. **세션 시작**을 누르면 보드 설정으로 세션이 열리고 Capture가 활성화된다.
2. **Capture** → 촬영 즉시 검증 파이프라인(교차점 검출 / 선명도(블러) / 명암·대비 / 화면 내 위치 / 크기)을 거친다.
   - 검출된 교차점이 그려진 이미지가 표시되고, **검증 통과 시에만 저장 버튼이 활성화**된다.
   - 실패하면 사유(예: "이미지가 흐립니다", "보드가 화면 가장자리에 잘렸습니다")가 표시되고 저장이 막힌다.
3. **저장** → 통과한 이미지를 세션 폴더에 누적 저장. 상태줄에 **수집 장수**와 **커버리지(N/27)**가 표시된다.
4. **포즈 다양성 유도**: 화면을 3×3 영역 × 3거리(근/중/원)로 나눠 커버리지를 추적한다.
   비슷한 구도를 반복하면 "이미 있는 구도" 안내가, 아직 안 찍은 구도는 "권장 미커버 구도"로 표시된다.
   구석·다양한 거리·기울기를 폭넓게 담을수록 캘리브레이션 품질이 좋아진다.

### 5.2 Extrinsic (다중 동기 카메라)

전제: 인접 카메라끼리만 화각이 겹치는 **링/배열형**. 카메라 간 상대 자세는 **인접 쌍 stereo
calibrate → 체이닝**으로 구한다(계산은 오프라인). 따라서 이 모드는 인접 쌍이 **동시에** 보드를
선명히 본 동기 촬영 세트를 모은다.

1. 여러 카메라 선택 → 전 카메라 라이브 프리뷰 + 동기 상태(최대편차/σ) 표시. **세션 시작**(필요 시 "링 배열"
   체크 → 양 끝 쌍 0-3 추가).
2. 보드를 특정 인접 쌍의 겹침 영역에 두고 **Sync Capture** → 동기 촬영 후 카메라별로 검증한다.
   - **인접 쌍의 두 카메라가 모두 통과**해야 그 shot이 저장 가능해진다(동시 검출 쌍이 표시됨).
   - 통과 시 저장, 미통과 시 카메라별 사유가 표시된다.
3. **저장** → shot 폴더에 **검증을 통과한(사용 가능) 카메라의 이미지만** 저장되고, 감지 실패/품질
   미달 카메라는 파일 없이 `detections.json`에 사유만 기록된다(어떤 이미지가 유효한지 명확히 구분).
   "쌍 커버리지"가 갱신되며 목표(쌍당 6장) 미달 쌍은 노란색으로 표시된다. 각 카메라 미리보기 아래에도
   `✓사용가능 / △품질미달·제외 / ✗미검출·제외`로 표시된다.
4. **스케일/월드 프레임**: 칸 크기(mm)가 미터 스케일을 준다. 카메라 간 상대 extrinsic만 구할 경우
   보드의 절대 위치는 불필요하며, 각 shot의 보드가 두 카메라의 공통 기준이 된다.

> 자세한 촬영 요령은 Extrinsic 패널의 "4-카메라 Extrinsic 촬영 가이드" 접기 항목 참조.

## 6. 저장 결과물

```
captures/
  20260708_153000/                 # Mode 1/2 촬영
    video0.jpg
    video1.jpg
    ...
    sync_report.json               # 다중 동기 촬영일 때만 생성
  calib/
    intrinsic/20260708_154500/
      board_config.json
      video0/frame_000.jpg, frame_001.jpg, ...
      report.json                  # 카메라별 [{file, metrics, descriptor}]
    extrinsic/20260708_160000/
      board_config.json
      shot_000/  (사용 가능 카메라 이미지만) video0.jpg,video1.jpg,...  detections.json
      shot_001/...
      report.json                  # shot별 saved_cameras·covered_pairs + 인접 쌍별 누적 장수(pair_counts)
```

- `board_config.json`: 세션 시작 시 입력한 보드 설정(종류/코너 수/칸·마커 mm/사전).
- intrinsic `report.json`: 카메라별 저장 프레임의 품질 지표와 포즈 descriptor(오프라인 캘리브레이션 입력).
- extrinsic `shot_XXX/detections.json`: 카메라별 검출 여부·교차점 수·**사용 가능(usable)** 여부·품질 지표
  (제외된 카메라는 사유 포함), 실제 저장된 카메라(`saved_cameras`), 동시 검출된 인접 쌍(`covered_pairs`),
  동기 통계(`sync_stats`).

`sync_report.json` 예시:

```json
{
  "per_camera": {"0": 0.0, "1": 1.23, "2": 0.87, "3": 3.41},
  "spread_ms": 3.41,
  "std_ms": 1.28,
  "ref_dev": 0
}
```

- `per_camera`: 카메라별 상대 타임스탬프(기준 카메라 대비, ms)
- `spread_ms`: 최대편차(최댓값 − 최솟값)
- `std_ms`: 표준편차
- `ref_dev`: 기준(가장 이른 타임스탬프) 카메라 번호

## 7. 참고

- "촬영 실패" / "동기 촬영 실패" toast가 뜨면 카메라 연결 상태를 확인하고 **카메라 재감지**를 눌러본다.
- 해상도를 바꾸면 라이브 스트림/동기 세션이 모두 재시작되므로 촬영 중이던 화면은 초기화된다.
- 파라미터(exposure/gain 등) 조정 UI는 아직 화면에 노출되지 않으며, 백엔드 API(`/api/params`)만 존재한다.
